# Voice-to-Voice Conversation Testing

Test complete voice conversations where both the user simulator _and_ the agent speak over multiple turns. This pattern is perfect for testing complex dialogues, multi-turn reasoning, and natural conversation flow.

<Callout type="info">
  This approach came out &apos;before&apos; OpenAI&apos;s real-time audio API
  and still works today.
</Callout>

## Prerequisites & Setup

**Common requirements:**

- **OPENAI_API_KEY** with access to `gpt-4o-audio-preview`

```bash
export OPENAI_API_KEY="sk-…"
```

**Language-specific setup:**

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

## Helper Utilities

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

- **[wrapJudgeForAudio](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/wrap-judge-for-audio.ts)** - Wraps a judge agent to automatically transcribe audio messages to text before evaluation. Required because judge agents can't process audio directly.

- **[encodeAudioToBase64](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-encoding.ts)** - Converts audio files to base64-encoded strings for transmission in messages.

- **[getFixturePath](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/fixture-utils.ts)** - Utility to resolve paths to test fixtures relative to the test directory.

- **[convertCoreMessagesToOpenAIMessages](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/convert-core-messages-to-openai.ts)** - Converts Scenario's CoreMessage format to OpenAI's ChatCompletion format, handling audio file detection.

- **[OpenAiVoiceAgent](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/openai-voice-agent.ts)** - Abstract base class that handles OpenAI voice API calls, message conversion, and response processing. Used by both the agent and user simulator.

- **[messageRoleReversal](https://github.com/langwatch/scenario/blob/main/javascript/src/agents/utils.ts)** - Swaps user ↔ assistant roles in messages (excluding tool calls) so the voice user simulator can speak as the user.

- **[saveConversationAudio](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-conversation.ts)** - Saves audio responses to files for debugging and analysis.

- **[concatenateWavFiles](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-conversation.ts)** - Combines multiple WAV files into a single conversation recording.

- **[getAudioSegments](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-conversation.ts)** - Extracts individual audio segments from a conversation for analysis.

</LanguageTabs.MarkdownTab>
<LanguageTabs.MarkdownTab language="python">

- **[wrap_judge_for_audio](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/judge_audio_wrapper.py)** - Wraps a judge agent to automatically transcribe audio messages to text before evaluation using OpenAI Whisper.

- **[OpenAiVoiceAgent](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/openai_voice_agent.py)** - Base class for voice-enabled agents. Handles OpenAI API calls, message conversion (forcing audio messages to `role='user'` per OpenAI's requirements), and response processing.

- **[save_conversation_audio](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/audio_helpers.py)** - Saves audio conversations to WAV files for debugging and analysis.

- **[concatenate_wav_files](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/audio_helpers.py)** - Combines multiple WAV files into a single continuous audio recording.

- **[reverse_roles](https://github.com/langwatch/scenario/blob/main/python/scenario/_utils/utils.py)** - Swaps user ↔ assistant roles in messages for user simulator agents.

</LanguageTabs.MarkdownTab>
</LanguageTabs>

These helpers handle the technical details like audio encoding, file management, message format conversion, and managing the OpenAI API's specific requirements for voice models.

:::warning
**Caveat**:

When the assistant responds with audio, OpenAI requires audio messages to use `role='user'`.

The helpers handle this automatically, but if needed you can pass `forceUserRole: true` (TypeScript) or `force_user_role=True` (Python) to `OpenAiVoiceAgent`.
:::
