# Audio → Audio Testing

Test agents that listen to audio input and reply with audio responses. This pattern is ideal for voice assistants, conversational AI, and any agent that needs to respond in a natural spoken voice.

## Prerequisites & Setup

**Common requirements:**

- **OPENAI_API_KEY** with access to `gpt-4o-audio-preview`

```bash
export OPENAI_API_KEY="sk-…"
```

**Language-specific setup:**

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

**Node.js ≥ 18** required for modern `fetch` API.

<br />

Install dependencies:

```bash
npm install scenario-sdk openai
```

</LanguageTabs.MarkdownTab>
<LanguageTabs.MarkdownTab language="python">

**Python ≥ 3.10** required for modern async/await.

<br />

Install dependencies:

```bash
uv pip install scenario-sdk openai
```

</LanguageTabs.MarkdownTab>
</LanguageTabs>

## Helper Utilities

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

- **[wrapJudgeForAudioTranscription](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/wrap-judge-for-audio-transcription.ts)** - Wraps a judge agent to automatically transcribe audio files to text before evaluation using OpenAI Whisper. Required because judge agents can't process audio directly.

- **[wrapJudgeForAudioExtraction](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/wrap-judge-for-audio-extraction.ts)** - Wraps a judge agent to extract text transcripts from audio messages that already contain transcripts (e.g., from realtime conversations).

- **[encodeAudioToBase64](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-encoding.ts)** - Converts audio files to base64-encoded strings for transmission in messages.

- **[getFixturePath](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/fixture-utils.ts)** - Utility to resolve paths to test fixtures relative to the test directory.

- **[convertCoreMessagesToOpenAIMessages](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/convert-core-messages-to-openai.ts)** - Converts Scenario's CoreMessage format to OpenAI's ChatCompletion format.

- **[OpenAiVoiceAgent](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/openai-voice-agent.ts)** - Base class that handles OpenAI voice API calls, message conversion, and response processing.

</LanguageTabs.MarkdownTab>
<LanguageTabs.MarkdownTab language="python">

- **[wrap_judge_for_audio](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/judge_audio_wrapper.py)** - Wraps a judge agent to automatically transcribe audio messages to text before evaluation using OpenAI Whisper.

- **[OpenAiVoiceAgent](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/openai_voice_agent.py)** - Base class for voice-enabled agents that handles API calls and message conversion.

- **[save_conversation_audio](https://github.com/langwatch/scenario/blob/main/python/examples/helpers/audio_helpers.py)** - Saves audio responses to files for debugging and analysis.

</LanguageTabs.MarkdownTab>
</LanguageTabs>

:::warning
**Caveat**:

When the assistant responds with audio, OpenAI requires audio messages to use `role='user'`.

The helpers handle this automatically, but if needed you can pass `forceUserRole: true` (TypeScript) or `force_user_role=True` (Python) to `OpenAiVoiceAgent`.
:::
