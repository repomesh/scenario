# Audio → Text Testing

Test agents that receive audio files (like WAV or MP3) and respond with textual answers. This pattern is ideal for transcription services, audio-based Q&A, or any agent that processes voice input but responds in text.

## Prerequisites & Setup

**Common requirements:**

- **OPENAI_API_KEY** with access to `gpt-4o-audio-preview`

```bash
export OPENAI_API_KEY="sk-…"
```

**Language-specific setup:**

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

**Node.js ≥ 18** required for modern `fetch` API.

<br />

Install dependencies:

```bash
npm install scenario-sdk openai
```

</LanguageTabs.MarkdownTab>
<LanguageTabs.MarkdownTab language="python">

**Python ≥ 3.10** required for modern async/await.

<br />

Install dependencies:

```bash
uv pip install scenario-sdk openai
```

</LanguageTabs.MarkdownTab>
</LanguageTabs>

## Helper Utilities

<LanguageTabs>
<LanguageTabs.MarkdownTab language="typescript">

- **[encodeAudioToBase64](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/audio-encoding.ts)** - Converts audio files to base64-encoded strings for transmission in messages.

- **[getFixturePath](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/fixture-utils.ts)** - Utility to resolve paths to test fixtures (like audio files) relative to the test directory.

- **[convertCoreMessagesToOpenAIMessages](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/helpers/convert-core-messages-to-openai.ts)** - Converts Scenario's CoreMessage format to OpenAI's ChatCompletion format, handling audio file detection and transformation.
