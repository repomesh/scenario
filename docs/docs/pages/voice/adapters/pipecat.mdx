# `PipecatAgentAdapter`

Drives a custom Pipecat bot. Connects to your bot's WebSocket endpoint (which
speaks the Twilio Media Streams binary protocol) or to a WebRTC signaling
endpoint.

## Constructor

:::code-group

```python [python]
import scenario

adapter = scenario.PipecatAgentAdapter(
    url="ws://localhost:8765/stream",
    audio_format="mulaw",
    sample_rate=8000,
)
```

```typescript [typescript]
import scenario from "@langwatch/scenario";

const adapter = scenario.pipecatAgent({
  url: "ws://localhost:8765/stream",
  audioFormat: "mulaw", // or "pcm16"
  sampleRate: 8000, // match your bot's config
});
```

:::

Other arguments: `signalingUrl` / `signaling_url` (for `transport: "webrtc"`),
`transport` (`"websocket"` default or `"webrtc"`), `streamSid` / `stream_sid`,
`callSid` / `call_sid`.

<Callout type="warning">
  **TypeScript: WebRTC mode is pending.**
  `scenario.pipecatAgent({ transport: "webrtc", signalingUrl })` constructs fine
  but raises `voice.PendingTransportError` at `connect()` until the SmallWebRTC
  transport lands. WebSocket mode (above) is fully shipped.
</Callout>
