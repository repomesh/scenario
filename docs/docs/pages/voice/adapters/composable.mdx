# `ComposableVoiceAgent`

Plug in a `STTProvider`, an LLM model string, and a TTS model string. Useful
when you want to test a hand-rolled voice stack against the same scenario
harness as the bundled adapters.

## Constructor

:::code-group

```python [python]
import scenario

agent = scenario.ComposableVoiceAgent(
    stt=scenario.ElevenLabsSTTProvider(api_key=...),
    llm="openai/gpt-5.4-mini",
    tts="elevenlabs/some-voice-id",
    system_prompt="You are a helpful assistant.",
)
```

```typescript [typescript]
import scenario, { voice } from "@langwatch/scenario";
import { openai } from "@ai-sdk/openai";

const agent = scenario.composableAgent({
  stt: new voice.OpenAISTTProvider(), // any STTProvider (e.g. voice.ElevenLabsSTTProvider)
  llm: openai("gpt-4.1-mini"),        // an AI-SDK LanguageModel
  tts: "openai/nova",                 // "<provider>/<voiceId>"
  systemPrompt: "You are a helpful assistant.",
});
```

:::

`stt` accepts any `STTProvider` implementation (`voice.OpenAISTTProvider`,
`voice.ElevenLabsSTTProvider`, or your own). Note `llm` differs by SDK: Python
takes a model string; TypeScript takes an AI-SDK `LanguageModel`.

## Capabilities

| streaming_transcripts | native_vad | dtmf | interruption | input_formats | output_formats |
| --- | --- | --- | --- | --- | --- |
| ✅ | ❌ | ❌ | ❌ | `pcm16/24000` | `pcm16/24000` |

VAD: Composable has no native VAD; the SDK falls back to `webrtcvad`. See
[Troubleshooting → VAD didn't fire](/voice/troubleshooting#vad-didnt-fire).

## Worked examples

- See the [ElevenLabs composable variant](/voice/adapters/elevenlabs#composable-variant--elevenlabsvoiceagent) for a working pattern in both SDKs.
- **TypeScript:** [`composable-stt-swap.test.ts`](https://github.com/langwatch/scenario/blob/main/javascript/examples/vitest/tests/voice/composable-stt-swap.test.ts) — swaps the judge's STT provider per run via `run(cfg, { voice: { stt } })`.
